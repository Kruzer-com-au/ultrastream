    - If using Google Fonts, use `next/font/google` with `display: 'swap'` and `preload: true` (this should already be set from Phase 1, but VERIFY and fix if missing)
    - If using local fonts, use `next/font/local` with the same options
    - Ensure font variables are applied to `<html>` or `<body>` className (not loaded via CSS @import which blocks render)
    - Confirm NO external font CSS links in `<head>` -- all fonts must go through next/font

    - Find the React Three Fiber hero scene component (likely `src/components/HeroScene.tsx` or similar from Phase 2)
    - Wrap the R3F scene import with `next/dynamic`:
      ```
      const HeroScene = dynamic(() => import('@/components/HeroScene'), {
        ssr: false,
        loading: () => <HeroSceneFallback />
      })
      ```
    - Create a lightweight `<HeroSceneFallback />` that matches the scene's dimensions with a dark gradient background and subtle CSS shimmer/pulse animation -- this prevents CLS and gives visual continuity while the 3D scene loads
    - Ensure `@react-three/fiber`, `@react-three/drei`, and `three` are NOT in the initial JS bundle -- they should only load when the dynamic import resolves
    - If there are other heavy 3D components below the fold, wrap those with `dynamic()` too

    **4. Image optimization audit:**
    - Scan all `<img>` tags in the codebase -- every image should use `next/image` (`<Image>`) component
    - For above-the-fold images (hero area): add `priority` prop (disables lazy loading, triggers preload)
    - For below-the-fold images: ensure `loading="lazy"` (default in next/image)
    - Add explicit `width` and `height` to all images to prevent CLS
    - If any images are in `public/` as PNG/JPG, next/image will auto-optimize them -- no manual conversion needed

    **5. Bundle analysis (verify, don't ship the analyzer):**
    - Run `npx @next/bundle-analyzer` or install temporarily to check bundle composition
    - Verify that `three`, `@react-three/fiber`, `@react-three/drei` are NOT in the main chunk
    - Verify that GSAP is loaded only where used (not globally bundled)
    - Check for any unexpectedly large dependencies and note them
    - Remove the analyzer dependency after checking (do not ship it)

    - Run `npx lighthouse http://localhost:3000 --output=json --output-path=./lighthouse-report.json --chrome-flags="--headless"` (or use the CLI if available)

    - If CLS > 0.1: identify shifting elements and add explicit dimensions
    - If TBT > 200ms: identify long tasks and defer non-critical JS

    **1. Plausible analytics provider (`src/components/Analytics.tsx`):**
    - Install `next-plausible` package: `npm install next-plausible`
    - Create a client component (`'use client'`) that wraps children with `<PlausibleProvider>`:
      ```
      import PlausibleProvider from 'next-plausible'

      export function AnalyticsProvider({ children }: { children: React.ReactNode }) {
        return (
          <PlausibleProvider
            domain="ultrastream.gg"
            customDomain={process.env.NEXT_PUBLIC_PLAUSIBLE_DOMAIN}
            selfHosted={!!process.env.NEXT_PUBLIC_PLAUSIBLE_DOMAIN}
            trackOutboundLinks
            trackFileDownloads
            enabled={process.env.NODE_ENV === 'production'}
          >
            {children}
          </PlausibleProvider>
        )
      }
      ```
    - The `domain` should fallback to "ultrastream.gg" but be overridable via env var `NEXT_PUBLIC_PLAUSIBLE_SITE`
    - `enabled` flag ensures analytics only fires in production (no noise during dev)

    **2. Custom event helpers (`src/lib/analytics.ts`):**
    - Export typed event tracking functions using `usePlausible` hook pattern:
      ```
      export type AnalyticsEvents = {
        'Scroll Depth': { depth: '25' | '50' | '75' | '100' }
        'Waitlist Form View': { location: string }
        'Waitlist Signup': { location: string }
        'CTA Click': { label: string; location: string }
      }
      ```
    - Export a `trackEvent` helper that calls `window.plausible` directly for non-component contexts:
      ```
      export function trackEvent<E extends keyof AnalyticsEvents>(
        event: E,
        props?: AnalyticsEvents[E]
      ) {
        if (typeof window !== 'undefined' && window.plausible) {
          window.plausible(event, { props })
        }
      }
      ```
    - Export a `useScrollDepth` hook that fires scroll depth events at 25%, 50%, 75%, 100% thresholds (fires each only once per page load using a ref to track fired thresholds)

    **3. Wire analytics into the app:**
    - In `src/app/layout.tsx`, wrap the app with `<AnalyticsProvider>`
    - In the main page (`src/app/page.tsx`), add the `useScrollDepth()` hook (or add it to a client wrapper component if page.tsx is a server component)
    - In the waitlist form component (from Phase 3), add:
      - `trackEvent('Waitlist Form View', { location })` when the form scrolls into viewport (use IntersectionObserver or existing scroll logic)
      - `trackEvent('Waitlist Signup', { location })` on successful form submission
    - In the sticky CTA component (from Phase 3), add:
      - `trackEvent('CTA Click', { label: 'Join the Revolution', location: 'sticky-bar' })` on click

    **4. Accessibility audit and fixes:**
    - Run `npx pa11y http://localhost:3000` or use Lighthouse accessibility audit
    - Fix the most common issues:
      - All images must have `alt` text (decorative images get `alt=""`, meaningful images get descriptive text)
      - All interactive elements (buttons, links, form inputs) must be keyboard-focusable and have visible focus styles (Tailwind: `focus-visible:ring-2 focus-visible:ring-accent`)
      - Form inputs must have associated `<label>` elements (the waitlist email input)
      - Color contrast ratios must meet AA (4.5:1 for normal text, 3:1 for large text) -- check neon text on dark backgrounds especially
      - Add `aria-label` to icon-only buttons (social links, close buttons)
      - Ensure the page has exactly one `<h1>` (the hero headline)
      - Add `role="main"` to `<main>` element or use semantic `<main>` tag
      - Skip navigation link: add a visually-hidden "Skip to content" link as the first focusable element in layout.tsx that jumps to `#main-content`
    - Target: Lighthouse Accessibility score >= 90
