**src/lib/scroll/scroll-trigger-setup.ts:**
Create a module that initializes the GSAP + Lenis integration. This runs ONCE at app startup.

```ts
import gsap from 'gsap';
import { ScrollTrigger } from 'gsap/ScrollTrigger';

gsap.registerPlugin(ScrollTrigger);

export function initScrollTrigger() {
  // ScrollTrigger defaults for the site
  ScrollTrigger.defaults({
    toggleActions: 'play none none reverse', // play on enter, reverse on leave-back
    markers: process.env.NODE_ENV === 'development', // visible markers in dev only
  });
}

export { ScrollTrigger };
```

Do NOT register SplitText here -- it's already registered in the Manifesto component from 02-01. Keep plugin registrations co-located with their usage.

**src/lib/scroll/lenis-provider.tsx:**
Create a React context provider that initializes Lenis and syncs it with GSAP ScrollTrigger.

```tsx
'use client';
```

Implementation:
1. Create `LenisContext` with the Lenis instance
2. In the provider component, initialize Lenis on mount:
   ```ts
   const lenis = new Lenis({
     duration: 1.2,        // scroll momentum duration
     easing: (t) => Math.min(1, 1.001 - Math.pow(2, -10 * t)), // exponential ease-out
     orientation: 'vertical',
     gestureOrientation: 'vertical',
     smoothWheel: true,
     touchMultiplier: 2,   // responsive touch scrolling
   });
   ```
3. Sync Lenis with GSAP ScrollTrigger:
   ```ts
   lenis.on('scroll', ScrollTrigger.update);
   gsap.ticker.add((time) => lenis.raf(time * 1000));
   gsap.ticker.lagSmoothing(0); // prevent GSAP lag compensation from fighting Lenis
   ```
4. Call `initScrollTrigger()` during provider mount
5. Export `useLenis()` hook for child components to access the Lenis instance
6. Clean up on unmount: `lenis.destroy()`, remove GSAP ticker callback
7. Respect `prefers-reduced-motion`: if user has reduced motion enabled, set `lenis.options.duration = 0` (instant scroll, no smoothing)

**CRITICAL:** Lenis and ScrollTrigger must share the same scroll position. The `lenis.on('scroll', ScrollTrigger.update)` line is what makes them work together. Without it, ScrollTrigger animations will be out of sync with the smooth scroll position.

**src/hooks/usePerformanceMonitor.ts:**
Create a performance monitoring hook and React context.

The hook should:
1. Track FPS using `requestAnimationFrame` loop (measure time between frames)
2. Calculate rolling average FPS over last 60 frames
3. Expose a `performanceTier` value: 'high' (>50fps), 'medium' (30-50fps), 'low' (<30fps)
4. On 'low': set a flag `shouldReduceAnimations: true`
5. On 'medium': set `shouldReduceParticles: true` (hint for 3D scene)
6. Check `prefers-reduced-motion` media query and force 'low' tier if true
7. In development mode (`NODE_ENV === 'development'`), render an FPS counter as a fixed overlay in the top-right corner showing current FPS, average FPS, and tier label. Style it as a small dark pill with monospace text. Hide in production.
8. Wrap in a `PerformanceProvider` context so any component can read the tier

Export:
```ts
export function PerformanceProvider({ children }: { children: React.ReactNode })
export function usePerformance(): { fps: number; tier: 'high' | 'medium' | 'low'; shouldReduceAnimations: boolean; shouldReduceParticles: boolean; }
```

**src/components/animation/ScrollReveal.tsx:**
Create a reusable wrapper component that reveals its children when scrolled into view.

Props:
```ts
interface ScrollRevealProps {
  children: React.ReactNode;
  preset?: 'fade-up' | 'fade-in' | 'scale-in' | 'slide-left' | 'slide-right' | 'blur-in';
  delay?: number;         // seconds
  duration?: number;      // seconds (default 0.8)
  stagger?: number;       // seconds between children (for lists)
  scrub?: boolean | number; // tie to scroll position (true = 1:1, number = smoothing)
  start?: string;         // ScrollTrigger start (default "top 85%")
  end?: string;           // ScrollTrigger end (default "top 20%")
  once?: boolean;         // only animate once, don't reverse (default true)
  className?: string;
}
```

Implementation:
1. Use `useGSAP` hook with a ref on the wrapper div
2. Map presets to GSAP `from` values:
   - `fade-up`: `{ opacity: 0, y: 60 }`
   - `fade-in`: `{ opacity: 0 }`
   - `scale-in`: `{ opacity: 0, scale: 0.85 }`
   - `slide-left`: `{ opacity: 0, x: -80 }`
   - `slide-right`: `{ opacity: 0, x: 80 }`
   - `blur-in`: `{ opacity: 0, filter: 'blur(10px)' }`
3. If `stagger` is set, target direct children instead of the wrapper
4. Check `usePerformance()` -- if `shouldReduceAnimations`, only do opacity fade regardless of preset
5. If `once` is true, set `toggleActions: 'play none none none'`
6. Use `gsap.from()` (not `gsap.to()`) so elements start in their animated state and resolve to their CSS-defined position

The component renders as a `div` (or `as` prop for semantic elements) that wraps children. It should NOT cause layout shift -- set initial styles via GSAP's `set()` method, not CSS classes (GSAP handles the initial state before ScrollTrigger fires).

**src/components/animation/ParallaxLayer.tsx:**
Create a parallax speed layer for depth effects.

Props:
```ts
interface ParallaxLayerProps {
  children: React.ReactNode;
  speed?: number;        // parallax multiplier (-1 to 1, negative = opposite direction)
  direction?: 'vertical' | 'horizontal';
  className?: string;
}
```

Implementation:
1. Use `useGSAP` with ScrollTrigger in `scrub: true` mode
2. Calculate y offset: `yPercent: speed * -50` (negative speed = moves up as you scroll down)
3. Trigger: the parent section, start: "top bottom", end: "bottom top" (active for the entire section visibility)
4. If `shouldReduceAnimations`, render children without parallax (still render, just no transform)
5. Set `will-change: transform` on the wrapper for GPU acceleration

This creates the depth illusion when multiple layers at different speeds are stacked within a section.

**src/components/animation/SectionTransition.tsx:**
Create cinematic section transition effects. This is the workhorse component for Phase 3 sections.

Props:
```ts
interface SectionTransitionProps {
  children: React.ReactNode;
  effect?: 'curtain-reveal' | 'scale-up' | 'clip-expand' | 'parallax-stack';
  className?: string;
}
```

Effects:
1. **`curtain-reveal`**: Content starts clipped (clipPath: `inset(100% 0 0 0)`) and reveals top-to-bottom as user scrolls, like a curtain lifting. Scrubbed to scroll position.

2. **`scale-up`**: Section starts at `scale(0.85)` and `opacity(0.3)`, scales to full size as it enters viewport center. Creates a "zooming in" effect. Scrubbed.

3. **`clip-expand`**: Content starts clipped in a small circle (`clipPath: circle(0% at 50% 50%)`) and expands to fill the section. Dramatic reveal for key content.

4. **`parallax-stack`**: The section has slight negative margin-top so it overlaps the previous section, creating a "stacking cards" effect. Previous section continues scrolling under this one. Use `z-index` to layer properly.

Each effect uses GSAP ScrollTrigger with `scrub: 1` (smooth 1-second lag behind scroll) for a cinematic, controlled feel. The trigger start/end positions should be carefully tuned:
- Start when the section's top hits the bottom of the viewport
- End when the section's top reaches 20% from the top of the viewport
- This gives a generous scroll range for the animation

For `shouldReduceAnimations`, fall back to simple `fade-in` for all effects.

Add `pin: false` explicitly -- do NOT pin sections. Pinning fights with Lenis and creates scroll jank.
