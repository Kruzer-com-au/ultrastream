**src/hooks/useMouseParallax.ts:**
Create a custom hook that tracks normalized mouse position (-1 to 1 on both axes) relative to the viewport center. Use `useEffect` with `mousemove` event listener. Return `{ x, y }` as a ref (not state -- avoid re-renders on every mouse move). Apply subtle lerp smoothing (factor ~0.05) using `requestAnimationFrame` for fluid motion. On mobile (no mouse), return `{ x: 0, y: 0 }` and add subtle gyroscope-based tilt via `DeviceOrientationEvent` if available.

**src/components/hero/HeroParticles.tsx:**
Create a particle field using `@react-three/drei`'s `Points` and `PointMaterial` (NOT instanced mesh -- Points is faster for thousands of particles). Generate 1500-2000 particles in a sphere distribution (radius ~8 units) using `Float32Array` for positions. Each particle should be small (size 0.015-0.03), color pulled from the design system's accent palette (electric blue/purple glow). Use `useFrame` to slowly rotate the entire particle group (rotation.y += 0.0003 per frame). Apply mouse parallax: shift the particle group position slightly based on `useMouseParallax` values (multiply by 0.3 for subtle effect). Add a slow "breathing" scale animation (sin wave, amplitude 0.05, period ~8 seconds) so particles gently pulse.

**src/components/hero/HeroGeometry.tsx:**
Create 4-6 floating geometric shapes that evoke power and rebellion -- NOT generic cubes. Use:
- 2x `IcosahedronGeometry` (crystalline polyhedra) at different scales (1.2 and 0.8)
- 1x `OctahedronGeometry` (diamond/shard shape, scale 0.6)
- 1x `TorusGeometry` (ring of power, scale 1.5, tube 0.02 -- thin and elegant)
- Optional: 1x custom `BufferGeometry` forming an angular shard/blade shape

Materials: Use `meshStandardMaterial` with metalness 0.9, roughness 0.1 (highly reflective metallic). Color: dark base (near-black #0a0a0f) with emissive glow using the brand accent colors (electric blue emissive for some, deep purple for others, one with red/gold). Emissive intensity: 0.3-0.6 (glow, not blast).

Position shapes scattered in 3D space around the camera (z: -2 to -6, x/y: -3 to 3). Each shape gets its own slow rotation animation in `useFrame` (different speeds and axes for organic feel). Apply mouse parallax per-shape with varying intensity (front shapes move more, back shapes less -- depth parallax).

Wrap each shape in `@react-three/drei`'s `Float` component for gentle bobbing (speed 1-2, floatIntensity 0.3-0.5).

**src/components/hero/HeroScene.tsx:**
Create the main R3F `Canvas` component. Configuration:
- `camera={{ position: [0, 0, 5], fov: 60 }}`
- `dpr={[1, 2]}` (device pixel ratio clamped for performance)
- `gl={{ antialias: true, alpha: true, powerPreference: 'high-performance' }}`
- Set `style={{ background: 'transparent' }}` so CSS gradient shows behind on edges

Scene setup inside Canvas:
- `ambientLight` intensity 0.1 (barely there -- scene should be dark)
- One `pointLight` at position [5, 5, 5] with intensity 0.8, color matching brand accent (electric blue)
- One `pointLight` at position [-3, -2, 4] with intensity 0.4, color deep purple
- One `spotLight` from above-center for dramatic top-down illumination on the logo area
- `fog` via `<fog attach="fog" args={['#000000', 5, 15]} />` for depth fade
- Render `<HeroParticles />` and `<HeroGeometry />`
- Use `@react-three/drei`'s `PerformanceMonitor` to auto-adjust DPR if FPS drops below 30

Do NOT use `OrbitControls` -- the camera is fixed, only elements move with mouse parallax. Do NOT use post-processing (Bloom, etc.) in this task -- it kills mobile performance. The scene must feel dark, atmospheric, and alive through lighting and motion alone.

The Canvas must be positioned absolute behind the text content (z-index layering: 3D scene behind, text overlay in front).

**src/hooks/useGPUDetect.ts:**
Create a hook that wraps `@react-three/drei`'s GPU detection. Since `useDetectGPU` from drei only works inside R3F Canvas, instead use the standalone `detect` function from `@pmndrs/detect-gpu`:

```
pnpm add @pmndrs/detect-gpu
```

The hook should:
1. Call `getGPUTier()` on mount (async)
2. Return `{ tier: number, isMobile: boolean, isLoading: boolean }`
3. Tier mapping: 0-1 = low (use fallback), 2 = medium (use R3F with reduced particles), 3 = high (full experience)
4. Also check `navigator.hardwareConcurrency < 4` as additional low-end signal
5. Cache result in a module-level variable so re-renders don't re-detect

**src/components/hero/HeroSection.tsx:**
Orchestrator component that assembles the complete hero section.

Logic:

4. If tier < 2: render `<HeroFallback />`
5. Always render `<LogoReveal />` and `<Manifesto />` as overlays on top of whichever background is shown
