**Zod validation schema `src/lib/validations/waitlist.ts`:**

**Supabase migration `supabase/migrations/001_waitlist.sql`:**
```sql
CREATE TABLE IF NOT EXISTS waitlist (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  email TEXT NOT NULL UNIQUE,
  created_at TIMESTAMPTZ DEFAULT now() NOT NULL,
  referral_source TEXT,
  ip_country TEXT
);

CREATE INDEX idx_waitlist_email ON waitlist(email);
CREATE INDEX idx_waitlist_created_at ON waitlist(created_at DESC);
```
Keep it minimal -- single email field for maximum conversion. No extra fields on the form.

**Email confirmation function `src/lib/email/confirmation.ts`:**
Export `sendConfirmationEmail(email: string): Promise<{ success: boolean; error?: string }>`. Uses the Resend SDK. Email content:
- From: process.env.RESEND_FROM_EMAIL (fallback: "ULTRASTREAM <noreply@ultrastream.gg>")
- Subject: "You're In. The Revolution Starts Now."
- HTML body: On-brand confirmation. Dark background, brand colors. Key copy: "Welcome to the rebellion, [just use 'warrior' or 'revolutionary' -- we don't have their name]. You've claimed your spot on the ULTRASTREAM waitlist. When we launch, you'll be among the first to experience a streaming platform built for creators and viewers -- not corporations. The villains' days are numbered." Include ULTRASTREAM logo/text, social links, and "Tell a friend" CTA. Keep HTML simple and email-client-compatible (table-based layout, inline styles).
- Handle errors gracefully -- if Resend fails, log the error but do NOT fail the signup (email is a bonus, not a gate).

**API route `src/app/api/waitlist/route.ts`:**
Export `POST` handler:
1. Parse and validate request body with `waitlistSchema.safeParse()`
2. If validation fails: return 400 with `{ error: "Invalid email address" }`
3. Create Supabase client using `createClient` with `NEXT_PUBLIC_SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` (server-side, not the anon key)
4. Insert email into `waitlist` table
5. If duplicate (unique constraint violation / Postgres error code 23505): return 409 with `{ error: "You're already on the list, revolutionary! We'll be in touch." }` -- friendly, not an error
6. If insert succeeds: call `sendConfirmationEmail(email)` -- fire and forget (don't await if it would slow the response; or await but don't fail on email error)
7. Query count of total waitlist entries: `SELECT count(*) FROM waitlist`
8. Return 201 with `{ success: true, message: "Welcome to the revolution!", count: totalCount }`

Also export `GET` handler that returns `{ count: totalCount }` for the social proof counter to poll.

**Rate limiting:** Add basic rate limiting via a simple in-memory Map (IP -> timestamp array). Limit to 5 signups per IP per hour. Return 429 if exceeded. This is not production-grade rate limiting but prevents basic abuse. Comment that a proper solution (Upstash Redis) should replace this for production.

Form submits via `fetch('/api/waitlist', { method: 'POST', body: JSON.stringify({ email }) })`. Handle all response codes (201, 400, 409, 429, 500).
